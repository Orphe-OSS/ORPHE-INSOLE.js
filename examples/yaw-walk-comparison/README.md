# Yaw Walk Comparison

ORPHE INSOLEを1台または2台接続し、閉じた経路を1周または2周歩いて、同じ受信データから次の3段階を比較する案内付きページです。

1. 旧パーサのスケール解釈をページ内で再現したyaw
2. Q14スケール修正と正規化を適用したyaw
3. 開始前の静止yawドリフトを学習して差し引いたyaw

旧状態は、修正済みQuaternionの各成分を0.5倍し、正規化せずEuler変換することで再現します。実機のSDKやファームウェアを旧版へ戻す必要はありません。

## 起動

リポジトリのルートでHTTPサーバーを起動します。

```bash
python3 -m http.server 8000
```

ChromeまたはEdgeで開きます。

```text
http://localhost:8000/examples/yaw-walk-comparison/
```

UIだけを確認する場合はシミュレーションモードを使います。

```text
http://localhost:8000/examples/yaw-walk-comparison/?sim=1
```

シミュレーション結果は実機評価に使用しないでください。

## 測定手順

1. 1台または左右2台を接続する。
2. 周回数と方向を選ぶ。
3. 開始位置と開始方向で「静止校正と記録準備」を押し、10秒静止する。
4. `READY`になったら「歩行を開始」を押す。
5. 指定した周回数を普通の速さで歩き、同じ位置・同じ向きへ戻る。
6. 「開始位置で終了」を押し、3秒静止する。
7. グラフと数値を確認し、CSVと結果JSONを保存する。

## データの扱い

- 計算とCSV生成はブラウザ内で完結します。
- 測定データをサーバーへ送信する処理はありません。
- `<meta name="robots" content="noindex,nofollow,noarchive">`を指定しています。
- `noindex`はアクセス制御ではありません。公開リポジトリ上のブランチURLを使う場合、URLを知らない第三者からのアクセスを暗号学的に防ぐものではありません。

## 評価上の注意

- 机周回の360°/720°は高精度な角度基準ではありません。
- 6軸IMUだけでは絶対方位を直接観測できないため、静止ドリフト補正後にも誤差が残る可能性があります。
- 通信欠損があると生gyro積分などの受信側計算へ影響します。Quaternionはデバイス内部で計算されますが、欠損区間の時系列はCSVに残りません。
- 静止ドリフト補正は比較評価用であり、SDKの標準Quaternion/Euler出力は変更しません。
