# ORPHE INSOLE Showcase — 企画書 / 仕様書

ORPHE INSOLE の「製品紹介・データ理解・使い方の入門」がひとつのWebアプリで完結する、オールインワンのショーケースexampleの企画書です。

- ステータス: **企画段階（実装前レビュー用）**
- 想定配置: `examples/showcase/`
- 作成日: 2026-06-12

---

## 1. 目的とターゲット

### 目的

ORPHE INSOLE を初めて触る人（研究者・開発者・展示来場者）が、このアプリ1つで以下を理解できる状態にする。

1. **製品が何か** — スペック概要、センサ構成、ORPHE COREとの違い
2. **何が取れるか** — 6ch圧力 / 加速度 / ジャイロ / クォータニオンを「動くグラフ・CG」で体感
3. **どう使うか** — 接続手順、ストリーミングモードの選択、各データを取得する最小コード

### ターゲットと利用シーン

| ターゲット | シーン | 重視する要素 |
|---|---|---|
| SDK導入を検討する開発者 | GitHub Pagesやローカルで閲覧 | コードスニペット、API対応表 |
| 研究者（非エンジニア含む） | 実機を接続して評価 | データの意味、可視化、L/R判定 |
| 展示・デモ来場者 | 実機装着デモ | 直感的なビジュアル、接続不要でも見られること |

## 3. コンセプト

> **「読める・動く・写せる」1ページツアー**

- **読める**: 接続しなくても製品スペックとデータの意味が文章+図で学べる（Safari/Firefoxでも閲覧可能）
- **動く**: INSOLEを接続すると、ページ内のすべての図がライブデータで動き出す
- **写せる**: 各セクションのコードスニペットをコピーすれば、そのまま自分のアプリの雛形になる

ページは縦スクロールのストーリー型（製品紹介LPのスタイル）とし、上から「製品概要 → 接続 → 圧力 → IMU → 姿勢3D → 応用（CoP/接地）→ 次のステップ」の順に体験が深まる構成にする。

### デモモード（重要な設計判断）

Web Bluetooth 非対応ブラウザや実機が無い環境のために、**記録済みサンプルデータ（歩行数十秒分のJSON）をループ再生するデモモード**を持つ。

- 初期状態はデモモードで全ビジュアルが動いている（「動かないページ」を見せない）
- 接続に成功した時点でライブデータに切り替え、ヘッダに LIVE / DEMO バッジを表示
- サンプルデータは `gotPress` / `gotConvertedAcc` 等と同じ形式で記録し、再生層はコールバック呼び出しを偽装する（可視化コードはライブ/デモを区別しない）

### 0. ヘッダ（固定バー）

- `buildInsoleToolkit()` を使用（接続トグル / 実測Hz / L/Rバッジ / バッテリー / モード設定）
- LIVE（緑）/ DEMO（グレー）バッジ。デモ中は「実機を接続すると全ての図がライブになります」とツールチップ
- セクションへのアンカーナビ（スクロールスパイ）

### 1. Hero: 製品概要

- 製品名、1行コンセプト（「6ch圧力 + IMU内蔵のインソール型IoTセンサー」）
- 静的コンテンツ。製品イメージは既存アセット（足型PNG）+ CSSで構成し、新規画像が用意できれば差し替え

### 2. スペック & センサ構成

- スペックカード（数値は実装時に公式仕様で最終確認する）:
  - センサ: 6ch圧力 + 3軸加速度 + 3軸ジャイロ + クォータニオン（オンボード融合）
  - サンプリング: 最大200Hz（モード1/3）/ 100Hz（モード4）
  - 通信: Bluetooth LE（Web Bluetooth対応）
  - バッテリー: 約24時間連続動作
  - 左右・装着位置: `device_information.mount_position` で取得可能
- インソールの図に6つの圧力センサ位置とIMU位置を注記したダイアグラム（SVG）
- ORPHE COREとの違い（CLAUDE.mdの比較表を要約した4行程度の表）

### 3. つないでみる

- 接続手順を3ステップ図解（電源ON → ヘッダのトグルON → デバイス選択）
- 対応ブラウザ表（Chrome/Edge ○、Safari/Firefox ×→デモモードで閲覧可）
- ストリーミングモード1/3/4の使い分け（CLAUDE.mdの判断ツリーを図化）。本アプリはモード4で動作
- 最小コードスニペット（`new OrpheInsole(0)` → `setup()` → `begin()`）

### 4. 圧力センサ（6ch）

- **左側**: 足型上ヒートマップ — hula-motion-sonifierの手法を踏襲し、足型画像上に6つの円を配置、値に応じて半径と色（青→赤）を変化。L/R判定に応じて足型画像を切替
- **右側**: 6chリアルタイム折れ線チャート（Chart.js、直近5秒、rAFスロットリング）
- 説明文: ADC生値であること、キャリブレーションの必要性、チャネル配置はモデル依存のためリマップ層を挟む推奨
- スニペット: `gotPress` の最小例

### 5. IMU: 加速度・ジャイロ

- 加速度 [G]（`gotConvertedAcc`）とジャイロ [deg/s]（`gotConvertedGyro`）の折れ線チャート2つ + 現在値の数値表示
- 説明文: 正規化値と実値の2系統のコールバックがあること、レンジは `getDeviceInformation()` で確認できること

### 6. 姿勢: クォータニオン → 3D（新規目玉）

- Three.js（CDN, r160系）で薄い箱型（or 押し出した足型シルエット）のインソール3Dモデルを表示し、`gotQuat` で `mesh.quaternion.set(x, y, z, w)`（座標系変換を挟む）
- 軸ヘルパー表示、ダブルクリックで姿勢リセット（基準クォータニオンのオフセット保存）
- 並べて `gotEuler` の pitch/roll/yaw をゲージ表示し、「クォータニオン→オイラー角」の関係を一目で示す
- 説明文: クォータニオンが得られるのはモード1/4のみであること
- モデルはプリミティブ（BoxGeometry/ExtrudeGeometry）で構成し、GLTF等の外部モデルファイルには依存しない

### 7. 応用: CoPと接地イベント

- CLAUDE.md の Pattern 1（接地検出ヒステリシス）と Pattern 3（CoP）をそのまま動かすライブデモ
- 足型上にCoP点 + 残像軌跡、接地/離地でインジケータ点滅（+任意で短いクリック音）
- 「圧力からイベントを自前で導出する」というINSOLEの基本パターンを体験させる（STEP_ANALYSIS非対応の代替であることを明記）
- スニペット: Pattern 1/3 のコード

- 他のexample（VISUALIZE / sensor-dashboard / hula-motion-sonifier / terminal）への用途別リンク
- API doc、GitHub、ORPHE-CORE.js へのリンク

## 5. 技術構成

| 項目 | 選定 | 理由 |
|---|---|---|
| 構成 | 静的HTML + JS数ファイル（ビルド工程なし） | 既存example規約に準拠。GitHub Pagesでそのまま公開可能 |
| UI | Bootstrap 5.2.2 + bootstrap-icons（CDN） | InsoleToolkit の前提と一致 |
| チャート | Chart.js 3.9.0（CDN） | VISUALIZEと同一バージョンで統一 |
| 3D | Three.js（CDN） | リポジトリ初導入。プリミティブのみ使用し軽量に |
| 接続UI | `src/InsoleToolkit.js` | 接続・再接続・モード変更UIを再実装しない |
| ストリーミング | モード4（100Hz, 全データ） | 全セクションが1モードで動く。デフォルトの提示も兼ねる |
| 描画 | データはバッファ、描画はrAFで約30fps | CLAUDE.md Pattern 5 準拠（必須） |

### ファイル構成案

```
examples/showcase/
├── index.html        # セクション構造 + 文章（説明文はHTML側に集約）
├── style.css         # セクションレイアウト、ヒートマップ、バッジ
├── app.js            # 接続・コールバック配線・デモ/ライブ切替・rAFループ
├── viz-pressure.js   # 足型ヒートマップ + 6chチャート + CoP/接地
├── viz-imu.js        # 加速度・ジャイロチャート
├── viz-3d.js         # Three.js 姿勢表示
├── demo-data.js      # 記録済みサンプルデータ（歩行 約30秒, ~100Hz）と再生機構
├── assets/           # 足型画像（hula-motion-sonifierから流用）、センサ配置SVG
└── README.md         # 概要・起動方法・スクリーンショット
```

設計方針: 可視化モジュールは `update(data)` を持つ独立オブジェクトとし、`app.js` がライブ（`got*`コールバック）とデモ（`demo-data.js` のプレイヤー）のどちらかから同じ形式で呼ぶ。1ファイル巨大化（hula-motion-sonifier方式）は避ける。

## 6. 非機能要件

- **パフォーマンス**: 100Hzコールバック内ではバッファ追加のみ。チャート更新・3D描画・DOM書き換えはrAF側に集約。長時間表示でもメモリが増えないようチャートデータは固定長リングバッファ
- **未接続体験**: 初回ロードから3秒以内にデモモードで全ビジュアルが動く。Web Bluetooth非対応ブラウザでは接続UIの代わりに案内文を表示
- **2台目対応**: 初版は1台（id 0）のみ。2台ダッシュボードは sensor-dashboard への導線で対応（スコープ外と明記）
- **言語**: 初版は日本語。文章はHTMLに集約し、将来 `data-i18n` 属性で英語化できる構造にしておく
- **依存**: CDNはBootstrap / Chart.js / Three.js の3つのみ。SDKは `../../src/` 参照（他example同様）

## 7. スコープ外（初版）

- 2台同時表示、左右バランス（→ sensor-dashboard）
- データ記録・CSVエクスポート（→ terminal / 将来example）
- 音による可聴化（→ hula-motion-sonifier）
- 歩行解析指標（ストライド等。FW非対応のため）

## 8. 開発ステップ案

1. **M1: 骨格** — index.html のセクション構造 + InsoleToolkit接続 + 文章流し込み（静的に完結）
2. **M2: 可視化** — 圧力ヒートマップ/6chチャート → IMUチャート → Three.js 3D の順に実装（ライブデータで動作確認）
3. **M3: デモモード** — 実機でサンプルデータを記録し `demo-data.js` 化、ライブ/デモ切替
4. **M4: 応用セクションと仕上げ** — CoP/接地デモ、コードスニペット、README、ルートREADMEからのリンク追加

各マイルストーンで `npm test` を通す（example追加のみならビルド不要だが、SDKに手を入れた場合は `npm run build` も実行）。

## 9. オープンな論点（レビューで決めたい）

1. **セクション3Dモデルの見た目**: プリミティブの箱で十分か、足型を押し出した形状まで作るか
2. **デモデータの収録**: 実機で歩行データを記録する作業が必要。既存の記録があれば流用したい
3. **example名**: `showcase` / `tour` / `getting-started` のいずれにするか（本書では `showcase` を仮置き）
4. **スペック数値の確定**: サンプリング周波数・バッテリー時間などの公称値は公式仕様書と照合が必要
