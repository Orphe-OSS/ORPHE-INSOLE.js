# ORPHE INSOLE Showcase — 企画書 / 仕様書

ORPHE INSOLE の「製品紹介・データ理解・使い方の入門」がひとつのWebアプリで完結する、オールインワンのショーケースexampleの仕様書です。

- ステータス: **初版実装済み**（デモ用歩行データは合成データ。実機収録CSVへの差し替え待ち）
- 配置: `examples/showcase/`
- 作成日: 2026-06-12

---

## 1. 目的とターゲット

### 目的

ORPHE INSOLE を初めて触る人（研究者・開発者・展示来場者）が、このアプリ1つで以下を理解できる状態にする。

1. **製品が何か** — スペック概要、センサ構成、ORPHE COREとの違い
2. **何が取れるか** — 6ch圧力 / 加速度 / ジャイロ / クォータニオンを「動くグラフ・CG」で体感
3. **どう使うか** — 接続手順、ストリーミングモードの選択、各データを取得する最小コード

### ターゲットと利用シーン

| ターゲット | シーン | 重視する要素 |
|---|---|---|
| SDK導入を検討する開発者 | GitHub Pagesやローカルで閲覧 | コードスニペット、API対応表 |
| 研究者（非エンジニア含む） | 実機を接続して評価 | データの意味、可視化、L/R判定、CSV記録 |
| 展示・デモ来場者 | 実機装着デモ | 直感的なビジュアル、接続不要でも見られること |

## 3. コンセプト

> **「読める・動く・写せる」1ページツアー**

- **読める**: 接続しなくても製品スペックとデータの意味が文章+図で学べる（Safari/Firefoxでも閲覧可能）
- **動く**: INSOLEを接続すると、ページ内のすべての図がライブデータで動き出す
- **写せる**: 各セクションのコードスニペットをコピーすれば、そのまま自分のアプリの雛形になる

### デモモード

Web Bluetooth 非対応ブラウザや実機が無い環境のために、**歩行データをループ再生するデモモード**を持つ。

- 初期状態はデモモードで全ビジュアルが動いている（「動かないページ」を見せない）
- ライブデータの受信が始まると自動でライブ表示に切り替え、ヘッダに LIVE / DEMO バッジを表示（ライブ途絶後は自動でデモに復帰）
- 初版は合成歩行データ（`demo-data.js`）。**本アプリのCSV記録機能で実機データを収録し、差し替える**（収録は利用者側で実施予定）
- 記録したCSVをその場で読み込んでデモ再生することも可能（再生層はライブと同じ経路でビジュアルを駆動する）

```
┌──────────────────────────────────────┐
│ 0. ヘッダ（固定）                       │ InsoleToolkit×2 + LIVE/DEMOバッジ + ナビ
├──────────────────────────────────────┤
│ 1. Hero: 製品概要                      │ 静的: リード文 + STORE/問い合わせボタン
├──────────────────────────────────────┤
│ 2. スペック & 購入導線                  │ 公式スペック表 + ORPHE STORE/評価キット
├──────────────────────────────────────┤
│ 3. つないでみる                        │ 接続手順 + 対応ブラウザ + モード解説
├──────────────────────────────────────┤
│ 4. 圧力センサ (6ch × 左右)             │ 足型ヒートマップ + CoP + 6chチャート ×2
├──────────────────────────────────────┤
│ 5. IMU: 加速度・ジャイロ (× 左右)       │ 折れ線チャート + 数値 ×2
├──────────────────────────────────────┤
│ 6. 姿勢: クォータニオン → 3D (× 左右)   │ p5.js+STL 左右2モデル + Euler + リセット
├──────────────────────────────────────┤
│ 7. データ記録 (CSV)                    │ 記録開始/停止/保存 + CSV再生（device列）
├──────────────────────────────────────┤
│ 8. 次のステップ                        │ 他exampleへのサムネイル付きカード
├──────────────────────────────────────┤
│ 9. 購入・お問い合わせ                   │ ORPHE STORE / 製品ページフォーム / Issues
└──────────────────────────────────────┘
```

**2台同時接続**: 左右最大2台（id 0/1）に対応し、1台でも動作する。圧力・IMU・Euler の各パネルと
3Dモデルはデバイスごとに持ち、`mount_position` の L/R 判定で画面上の並び（左足=左）に自動で揃える。
デモ再生も左右2足分（右足は半周期ずらした合成データ）を駆動する。

**購入導線**: Hero・スペック横・最終セクションの3箇所に [ORPHE STORE](https://shop.orphe.io/) と
[製品ページのお問い合わせフォーム](https://orphe.io/insole)（評価キット・共同開発の相談）へのリンクを置く。
ORPHE CORE との比較表は掲載しない（決定済み）。

### 0. ヘッダ（固定バー）

- `buildInsoleToolkit()` を使用（接続トグル / 実測Hz / L/Rバッジ / バッテリー / **モード設定ギア**）
- ストリーミングモードの切替はツールキットのギアから行う（決定済み）。アプリ側は現在のモードに追従し、配信されないデータのセクションには注記バッジを表示する（例: モード3ではクォータニオン欄に「このモードでは配信されません」）
- LIVE（緑）/ DEMO（グレー）バッジ
- セクションへのアンカーナビ

### 2. スペック & センサ構成（公式数値・確定）

| 項目 | 値 |
|---|---|
| サイズ (US men) | 5, 5.5, 6, 8.5, 9, 9.5 |
| 重量 | 約87g（片足） |
| モーションセンサ | 3軸加速度 + 3軸ジャイロ |
| 圧力センサ | 片足6点 |
| 加速度レンジ | ±16G |
| ジャイロレンジ | ±2000dps |
| 圧力レンジ | 0N–148N |
| サンプリングレート | 100Hz / 200Hz |
| 接続 | Bluetooth 5.2 |
| 連続動作時間 | 24時間 |
| 充電 | マグネットコネクタ / ワイヤレス充電対応 |

### 4. 圧力センサ（6ch）

- 足型画像（hula-motion-sonifierのアセット）上に6つのセンサ円を配置し、値に応じて色（青→赤）を変化。CoP（圧力中心）も同じマップ上に点+軌跡で表示（生データからの単純計算のみ。歩行イベント検出は行わない）
- 6chリアルタイム折れ線チャート（Chart.js、rAFスロットリング）
- L/R判定（mount_position）に応じて足型画像とセンサ座標を切替
- 説明文: ADC生値であること、キャリブレーションの必要性
- スニペット: `gotPress` の最小例

### 6. 姿勢: クォータニオン → 3D

- **ORPHE-CORE.js examples/VIEW をほぼ移植**: p5.js(WEBGL) + toxiclibs Quaternion + STL靴モデル（`assets/models/orphe_shoeL3.stl` / `orphe_shoeR3.stl` をCOREリポジトリからコピー）
- L/R判定に応じて表示モデルを切替
- 姿勢リセットボタン（基準クォータニオンの共役を掛けるオフセット方式。INSOLE SDKにはCOREの `resetMotionSensorAttitude()` が無いためアプリ側で実装）
- 並べて Euler (pitch/roll/yaw) をゲージ+数値で表示
- 注記: クォータニオンはモード1/4のみ

### 7. データ記録（CSV）

- 記録開始/停止、行数・経過時間表示、CSVダウンロード（接続中の全デバイスを1ファイルに記録）
- 列: `device, timestamp, serial_number, press0..5, acc_x/y/z [G], gyro_x/y/z [dps], quat_w/x/y/z, euler_pitch/roll/yaw [rad]`（モードにより欠ける列は空欄、device 列の無い旧形式CSVも読込可）
- 行の確定タイミング: モード3/4は各サンプルの最後に呼ばれる `gotPress`、モード1は `gotEuler`（SDKのコールバック順に基づく）
- 記録したCSVをファイル入力から読み込み、デモモードとしてループ再生できる（収録データの確認とデモデータ差し替えの動線）

## 5. 技術構成

| 項目 | 選定 | 理由 |
|---|---|---|
| 構成 | 静的HTML + JS数ファイル（ビルド工程なし） | 既存example規約に準拠 |
| UI | Bootstrap 5.2.2 + bootstrap-icons（CDN） | InsoleToolkit の前提と一致 |
| チャート | Chart.js 3.9.0（CDN） | VISUALIZEと同一バージョン |
| 3D | p5.js + toxiclibs（CDN）+ STLモデル同梱 | CORE VIEW からの移植（決定済み） |
| 接続UI | `src/InsoleToolkit.js` | モード切替UI込みで再利用 |
| ストリーミング | デフォルトはモード4。ツールキットのギアで1/3/4へ切替可能 | アプリは全モードに追従（決定済み） |
| 描画 | データはバッファ、描画はrAFで約30fps | CLAUDE.md Pattern 5 準拠 |

### ファイル構成

```
examples/showcase/
├── index.html        # セクション構造 + 説明文 + スニペット
├── style.css         # レイアウト、足型マップ、バッジ
├── app.js            # 接続・コールバック配線・LIVE/DEMO切替・CSV記録・rAFループ
├── viz-pressure.js   # 足型ヒートマップ + CoP + 6chチャート
├── viz-imu.js        # 加速度・ジャイロチャート
├── viz-3d.js         # p5.js + STL 姿勢表示（VIEW移植）
├── demo-data.js      # 合成歩行データ生成 + CSVパーサ
├── assets/
│   ├── orphe-insole-left.png / -right.png   # hula-motion-sonifier から流用
│   └── models/orphe_shoeL3.stl / orphe_shoeR3.stl  # ORPHE-CORE.js から流用
├── SPEC.md           # 本書
└── README.md         # 起動方法
```

データフロー: ライブ（`got*` コールバック）とデモ（再生プレイヤー）はどちらも同じ `dispatchFrame()` に1サンプル分のフレームを渡し、可視化モジュールはデータ源を区別しない。

## 6. 非機能要件

- 100Hz/200Hzコールバック内ではバッファ追加のみ。チャート・3D・DOM更新はrAF側
- 初回ロードから数秒以内にデモモードで全ビジュアルが動く
- Web Bluetooth非対応ブラウザでは接続トグルの代わりに案内を表示（閲覧は可能）
- STLモデルの読み込みに fetch を使うため、`file://` ではなくHTTPサーバ経由での閲覧を推奨（README記載）

## 7. スコープ外（初版）

- 歩行イベント検出・歩容解析指標（INSOLE.js / FW の歩容解析対応待ち。対応後に追補）
- 音による可聴化（→ hula-motion-sonifier）
