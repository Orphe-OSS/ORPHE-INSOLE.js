GESTURE LAB

RECORD RAW SENSOR DATA PER GESTURE → CSV
INSTRUMENT
1. GESTURE LABEL — 録るジェスチャを選択
2. RECORD — 開始→ジェスチャを5〜10回繰り返し→停止（SPACEでも操作可）
記録対象: — IDLE LABEL: — D0 — / D1 —
LIVE MONITOR0 Hz
PRESS 6CH (D0 solid / D1 dim)
|ACC|-1 [G] & GYRO XYZ [dps/1000]
3. TAKES0 TAKES / 0 ROWS
#LABELDURROWSDEV
RECORDING GUIDE
推奨手順: 各ラベルにつき1テイク、1テイク内で同じジェスチャを5〜10回、間に1秒程度の静止を挟む。
記録対象: 接続中のINSOLEを自動で全部記録します（選択不要）。2台接続で片手だけジェスチャした場合も、CSVの device / side 列と動きの大きさで解析側が判別するので問題ありません。
idle_hold（手で持って静止）と handling_noise（持ち替え・歩き回りなどの誤爆源）も必ず録ると閾値設計が正確になります。
fsr_sweep: どのFSRがch何番かは知らなくてOK。マークのあるFSRを適当な順に1つずつ「1秒押す→離す→1秒空ける」で全部押してください（反応chはデータから自動判別します）。
yaw_spin: インソールをお盆のように水平に持ったまま、レコードを回すように左右にクイックイッと回転させるジェスチャ（つま先が左右に振れる動き。上下には振らない）。
左右両方のINSOLEで録る場合は両方接続したまま同時録音でOK（行に device / side が入ります）。
エクスポートしたCSVはこのリポジトリの examples/music-shoe/recordings/ に置いてください — 解析してしきい値・判定アルゴリズムを作り込みます。
記録は streaming mode 4（100Hz）: 圧力＋IMUに加えてクォータニオンも記録されるため、姿勢（Euler）をオフラインで解析できます。
CSV列: take, label, device, side, ms(受信相対時刻), timestamp(センサ), serial, packet, p0..p5(ADC生値), ax,ay,az[G], gx,gy,gz[dps], qw,qx,qy,qz
