# クォータニオン・スケール修正 検証レポート

対象: ORPHE-INSOLE.js v1.2.1

## 結論と根拠

- 本PRでは SENSOR_VALUES header 50 / 56 の quaternion を signed Q14（`1.0 = 16384`）としてデコードする。
- acc / gyro の正規化コールバックは後方互換のため raw int16 / 32768 を維持する。
- `gotConvertedGyro` はセンサーのレンジ別感度を使って deg/s に変換する。
- Euler 角は quaternion を単位ノルムへ正規化してから計算する。
- BLEペイロード仕様で quaternion の Q14 スケールと、gyro がセンサーの raw int16 であることを確認した。

提供されたフィールドCSVでは、全サンプルの quaternion ノルムが約0.500だった。旧SDKの `/32768` デコード値と一致し、行ごとに正規化すると yaw の約±20°の圧縮が解消して約±180°の範囲が復元した。

> FWバージョンが異なる実機でも quaternion norm が約1.0になることを、実機検証ページで確認する。

## 既存CSVの救済手順

旧SDKで保存された quaternion は4成分が同じ比率で小さくなっているため、データは廃棄不要です。各行の `quat_w, quat_x, quat_y, quat_z` をその行のノルムで割り、長さ1へ正規化してください。ノルムが約0.5なら4成分を一律2倍することと同じです。その後、`euler_pitch, euler_roll, euler_yaw` は補正後の quaternion から再計算してください。Euler列そのものを2倍してはいけません。ノルム0または非有限値の行は無効として除外してください。
